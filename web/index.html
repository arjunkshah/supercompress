Agent memory layer · open source

Your agent forgets at turn 4. We fix that.

Every turn, agents gather web search + app data. Context explodes — slower, pricier, wrong answers. SuperCompress trims what matters before inference. One line of code. ~65% KV savings.

01 · The problem

      
More tools = more context = agents break

      

        Turn 1 feels magic. Turn 4 the model forgets your PRs, hallucinates, or times out.
        Not a smarter prompt — a memory problem.
      

      

        Turn 1→
        Turn 2→
        Turn 3→
        Turn 4 💥
      

      

        SuperCompress runs between gather and inference — every turn, automatically.
      

    


    

      
02 · Full stack (not a library bolt-on)

      
Each sponsor has a job

      
We don't checklist integrations. Each one creates or solves part of the turn-4 problem.

      

        Tavily→
        Composio→
        SuperCompress→
        Nebius→
        Composio
      

      

        

          
Tavily

          
Pours in live web research — verbose, redundant, huge.

          
Creates the bloat SuperCompress is built to cut.

        

        

          
Composio

          
GitHub, Gmail, Slack — gather state + execute tool calls each turn.

          
Multi-turn trap: every action adds more tokens back.

        

        

          
SuperCompress

          
Learned eviction policy. Keeps PR #42, drops duplicate noise.

          
The only step that shrinks context before the model reads it.

        

        

          
Nebius

          
Token Factory inference on compressed context + tool calling.

          
Proves the agent still reasons — cheaper, faster, fits KV cache.

        

        

          
OpenClaw

          
Runtime hook via SKILL.md + webhook. Long sessions, many turns.

          
Where turn 4 actually happens in production.

        

        

          
You

          
compress_for_turn() or supercompress loop — ship in one afternoon.

          
Drop into OpenClaw, Codex loops, any agent stack.

        

      

    


    

      
03 · Install

      
Live in 60 seconds

      
No API keys needed for the demo. Add yours for the full stack.

      

        
# clone + setup

        
git clone https://github.com/arjunkshah/supercompress.git

        
cd supercompress && ./setup.sh

        

        
# the demos judges (and Twitter) want to see

        
./bin/supercompress turns    # turn 4 problem visualized

        
./bin/supercompress loop     # full sponsor stack

        

        
# or pip install into your agent

        
pip install git+https://github.com/arjunkshah/supercompress.git
